# Триангуляционный фреймворк для архитектурных решений + соответствие гипотез выбранным методам

Документ консолидирует развёрнутую методологию проверки гипотез факторного эксперимента: семь статистических инструментов на простом языке, соответствие 4 инструментов 5 гипотезам, три согласованных термина вердикта, триангуляционный фреймворк для принятия архитектурных решений на условно подтверждённых гипотезах и таблицу «гипотеза → выбранный статистический метод → обоснование».

## Семь статистических инструментов на простом языке

| Инструмент | Что отвечает | Простой смысл |
|---|---|---|
| **Парный критерий Уилкоксона** | «Есть ли разница между двумя конфигурациями на одних и тех же задачах?» | Сравниваем стенды попарно (задача-к-задаче), а не «в среднем» — устраняет различия по сложности задач. Применяется к H1, H2a, H2b, H3, H4. |
| ***p*-значение** | «Какова вероятность увидеть такую же или более заметную разницу случайно, если её на самом деле нет?» | Чем меньше — тем меньше шанс, что наблюдаемое — совпадение. В работе порог 0,01 (вместо 0,05) — с поправкой на 5 параллельных гипотез. |
| ***d* Коэна (*d*_h, *d*_z)** | «Насколько разница велика, не только статистически значима?» | Размер эффекта в стандартных отклонениях; ≥ 0,8 — крупный, ≥ 0,5 — средний, ≥ 0,2 — малый. *d*_h — для пропорций, *d*_z — для парных разностей. |
| **Реализованная мощность (1 − β)** | «Если эффект существует, поймает ли его текущий тест?» | 0,99 — почти наверняка поймает; 0,04 — почти наверняка пропустит. Низкая мощность мелких эффектов на малой выборке — ограничение, а не ошибка. |
| **Поправка Холма (Бонферрони)** | «Как бороться с тем, что 5 гипотез разом — это 5 шансов на ложное открытие?» | Делает порог *p* строже: при 5 гипотезах общий порог 0,05 / 5 = 0,01. Холм мягче Бонферрони: у самой малой *p* порог 0,01, у самой большой — 0,05. |
| **TOST (Two One-Sided Tests)** | «Как доказать, что две вещи равны (а не просто что разница не доказана)?» | Обычный тест отвечает «разница есть/нет»; TOST отвечает «разница достаточно мала для паритета». Применяется только к H4. |
| **Bootstrap (бутстреп)** | «Какова прямая вероятность того, что эффект > 0, если данных мало для классического теста?» | 10 000 раз пересэмплируем выборку с возвратом и смотрим долю вариантов с нужным эффектом. Даёт «эмпирическую вероятность направления». |

## Соответствие гипотез методам — 4 инструмента под 5 гипотез

| Гипотеза | Тип вопроса | Размер эффекта | Основной инструмент | Дополнительные |
|:-:|---|---|---|---|
| **H1** | различие («Помогает ли разделение работы на роли?») | крупный (*d*_h = 1,98) | парный критерий Уилкоксона + Холм | — |
| **H2a** | различие (по знаку) | малый отрицательный (*d*_h = −0,17) | парный критерий Уилкоксона + Холм | — |
| **H2b** | различие (взаимодействие) | малый положительный (*d*_z = 0,26) | one-sample Wilcoxon на контрасте + Холм | bootstrap, стратификация, согласованность с H3 |
| **H3** | различие (3 пары) | средний (*d*_z = 0,79) | **mixed-effects модель с random intercept на задаче (primary)** + Холм | cluster-bootstrap, Fisher's combined *p*, стек one-sample Wilcoxon (summary) |
| **H4** | эквивалентность | малый (*d*_h = −0,35) | **TOST + 95 % ДИ Уилсона** | bootstrap, триангуляция метрик, стратификация |

## Два уровня подтверждения гипотез

Три согласованных термина для вердикта по каждой из 5 гипотез эксперимента:

- **«Формально подтверждена»** — точечная оценка превышает порог, статистическая значимость с поправкой Холма достигнута (*p* < α_adj), реализованная мощность не ниже 0,80. Все три условия подтверждения выполнены одновременно.
- **«Подтверждена условно по точечной оценке»** — точечная оценка превышает порог, знак эффекта согласован с теоретическим прогнозом, но формальная мощность не достигнута из-за ограниченной выборки *n* = 15 единиц теста. Вероятность ошибки II рода зафиксирована явно; для строгого подтверждения требуется расширение выборки. При строгом прочтении гипотеза с таким вердиктом формально не подтверждена — её допустимо трактовать как «не подтверждённую, но и не опровергнутую». Отдельный термин введён, чтобы отличать согласие знака эффекта с прогнозом от доказанного результата.
- **«Опровергнута»** — точечная оценка ниже порога либо имеет знак, противоположный гипотезе; ошибка I рода контролируется на уровне α_adj. Опровержение интерпретируется как самостоятельный научный результат.

Термины применяются единообразно ко всем пяти вердиктам факторного эксперимента и в сводной таблице вердиктов.

## Триангуляционный фреймворк для архитектурного решения

Когда вердикт «условно по точечной оценке» используется как основание для архитектурного выбора (производственного решения проекта), дополнительно проверяются четыре условия:

1. **Точечная оценка превышает pre-registered порог** (порог зафиксирован до проведения эксперимента — Nosek et al., 2018).
2. **Знак эффекта согласован с теоретическим прогнозом**, опирающимся на формально подтверждённую смежную гипотезу либо на формально установленный TOST non-inferiority.
3. **Bootstrap-вероятность направления эффекта** (P(эффект > 0) или P(эффект > порога)) — выше принятого порога достоверности (в работе — 0,80).
4. **Независимый признак** (стратификация по сложности задач, согласованность с формально подтверждённой смежной гипотезой, триангуляция метрик) подтверждает направление.

Сходимость всех четырёх условий — основание для архитектурного решения в условиях недостаточной формальной мощности двустороннего теста. Если хотя бы одно условие не выполнено — гипотеза остаётся как направление подтверждающего исследования, и архитектурное решение откладывается до расширения выборки.

## Применение чек-листа к условным гипотезам H2b и H4

| Условие | H2b (синергия RAG × граф) | H4 (паритет on-prem ↔ cloud) |
|---|---|---|
| 1. Точечная оценка выше порога | +9,33 п.п. > +5 п.п. ✓ | 84,85 % > 75 % ✓ |
| 2. Знак согласован с теоретическим прогнозом | синергия > 0; опирается на формально подтверждённую H3 ✓ | паритет > 75 %; опирается на формально установленный TOST non-inferiority 0,65 ✓ |
| 3. Bootstrap-вероятность направления > 0,80 | P(эффект > 0) = 0,845 ✓ | P(паритет ≥ 0,75) = 0,806 ✓ |
| 4. Независимый признак | монотонная дозозависимость по 3 стратам сложности (простые −4,0 → средние +8,0 → сложные +24,0 п.п.) ✓ | триангуляция 3 метрик (Pass@1 / Completeness / TimeToSuccess > 75 %) + стратификация 3 страт (92,0 / 82,6 / 77,8 % > 75 %) ✓ |
| **Итог** | **все 4 ✓ — архитектурное решение принимается** | **все 4 ✓ — архитектурное решение принимается** |

## Соответствие гипотез выбранным методам

Для каждой гипотезы выбран метод, обоснованный конкретным свойством данных или задачи. Методы не дублируют друг друга и совместно покрывают все необходимые проверки.

| Гипотеза | Выбранный метод | Обоснование выбора |
|:-:|---|---|
| **H1** (multi vs single, парное) | Wilcoxon signed-rank по 15 задачам (среднее по 5 повторам); 95 % ДИ Уилсона; *d*_h по пропорциям; поправка Холма | Парный непараметрический тест — устойчив на малой выборке без допущения нормальности; Уилсон даёт корректный ДИ для долей вблизи границ; *d*_h по арксинус-преобразованию — стандарт Cohen 1988 для пропорций; Холм — последовательная поправка на семейство 5 гипотез с большей мощностью, чем Bonferroni |
| **H2a** (RAG в изоляции, парное) | Wilcoxon signed-rank по 15 задачам; *d*_h; Холм | Та же логика, что и H1: одно сравнение двух конфигураций по одному и тому же набору задач; парный дизайн уменьшает дисперсию между задачами |
| **H2b** (RAG × Graph interaction) | One-sample Wilcoxon на контрасте per-task разностей `(E−D) − (G−B)` по 15 задачам; *d*_z; Холм | Эффект взаимодействия вычисляется методом разности разностей; один контраст на задачу превращает interaction в одну выборку парных разностей — корректно проверяется one-sample-вариантом Уилкоксона |
| **H3** (граф, среднее по 3 парам RAG = 0/3/5) | **Mixed-effects модель** `Δ ~ 1 + C(rag) + (1 | task_id)` (primary; учитывает кластеризацию 45 разностей по 15 задачам) + cluster-bootstrap по задачам (10 000 ресэмплов) + Fisher's combined p по 3 раздельным парным Wilcoxon; one-sample Wilcoxon на 45 разностях — summary-индикатор консистентности; *d*_h и *d*_z; Холм | 45 (task, RAG)-разностей кластеризованы по 15 задачам (3 RAG-условия на задачу); ICC = 0,74; mixed-effects модель — корректная инферентная оценка с учётом кластеризации; cluster-bootstrap независимо подтверждает направление; Fisher's combined p объединяет три раздельные парные пары (B→G, C→I, D→E); summary one-sample Wilcoxon (p = 5,8 × 10⁻⁵) показан как индикатор устойчивости знака по всем 45 конфигурациям, но завышен из-за допущения независимости |
| **H4** (паритет cloud vs on-prem) | Wilcoxon signed-rank по 15 задачам; точечная оценка паритета Pass@1(H)/Pass@1(G); 95 % ДИ Уилсона; *d*_h; Холм; **TOST для non-inferiority**; bootstrap для прямой вероятности направления | Парный тест по тем же 15 задачам устраняет различия сложности; *p* > α_adj согласуется с гипотезой паритета; TOST — корректный инструмент для проверки эквивалентности (двусторонний тест разности методологически неполон в задаче паритета); bootstrap даёт P(паритет ≥ 0,75) — прямую меру достоверности направления |

## Разграничение ролей инструментов

- **Wilcoxon signed-rank** — основной тест для парного сравнения двух конфигураций (H1, H2a, H2b, H4); для H3 — summary-индикатор консистентности по стеку 45 разностей, не primary.
- **Mixed-effects модель** с random intercept по задаче — primary тест для H3 с учётом кластеризации 45 разностей по 15 задачам (см. `docs/technical/compute_h3_mixed.py`, ICC = 0,74).
- **Cluster-bootstrap по задачам** — независимая проверка направления эффекта H3 без допущения нормальности распределения.
- **TOST** — корректный инструмент для гипотезы эквивалентности (только H4).
- **Bootstrap** — прямая вероятность направления эффекта без зависимости от дихотомии «*p* < α» (H2b, H4).
- **Поправка Холма** — контроль FWER на семействе 5 гипотез на верхнем уровне.

Инструменты не подменяют друг друга: каждый отвечает на свой вопрос.
