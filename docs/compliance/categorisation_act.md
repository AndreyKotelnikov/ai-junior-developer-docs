# Акт категорирования объекта КИИ — шаблон

**Назначение**: шаблон акта категорирования объекта критической информационной инфраструктуры (КИИ) для системы AI Junior Developer, развёрнутой в контуре КИИ-оператора. Документ задаёт структуру акта; конкретные значения заполняются комиссией по категорированию субъекта КИИ.
**Нормативная база**: 187-ФЗ «О безопасности КИИ» ст. 7; Постановление Правительства РФ № 127 от 08.02.2018 (правила категорирования); Приказы ФСТЭК № 235, № 239.
**Связанный документ**:
- [`compliance-matrix.md`](compliance-matrix.md) — матрица соответствия 152-ФЗ / 187-ФЗ / ФСТЭК.

> **Статус.** Система **технологически готова** к категорированию: документация соответствует III категории значимости. Формальная процедура категорирования и регистрация в реестре значимых объектов КИИ ФСТЭК России выполняются субъектом КИИ — задача R15.

---

## 1. Сведения об объекте КИИ

| Поле | Значение (заполняется субъектом КИИ) |
|---|---|
| Наименование объекта КИИ | _AI Junior Developer (мультиагентная система доработки .NET-кода)_ |
| Субъект КИИ | _указывается организацией_ |
| Сфера функционирования | _банковская сфера / оборона / госуправление — по принадлежности субъекта_ |
| Тип объекта | информационная система |
| Назначение | автоматизированная доработка enterprise-приложений на стеке ASP.NET Core + EF Core |

---

## 2. Перечень критических процессов

| № | Процесс | Критичность |
|:-:|---|---|
| 1 | Журналирование LLM-вызовов в append-only `pipeline_audit` | обеспечивает учёт и контроль целостности |
| 2 | Контроль человека над ИИ (`ApprovalGate`) | предотвращает неконтролируемые изменения кода |
| 3 | Air-gapped развёртывание (Ollama + локальный PostgreSQL) | исключает исходящий трафик из контура |

---

## 3. Оценка показателей значимости

Категорирование выполняется по 5 группам показателей (Постановление Правительства РФ № 127, приложение). Для системы AI Junior Developer в типовом контуре КИИ-оператора:

| Группа показателей | Оценка | Обоснование |
|---|---|---|
| Социальная значимость | не применяется / низкая | система не обслуживает граждан напрямую |
| Политическая значимость | низкая | вспомогательный инструмент разработки |
| Экономическая значимость | низкая — средняя | определяется ценностью дорабатываемого приложения |
| Экологическая значимость | не применяется | — |
| Значимость для обороны, безопасности и правопорядка | определяется субъектом КИИ | зависит от принадлежности субъекта |

**Предлагаемая категория значимости: III (третья, низшая)** — для типового контура. Итоговая категория устанавливается комиссией субъекта КИИ.

---

## 4. Меры защиты, реализованные в системе

| Мера | Технический механизм | Verifiable артефакт |
|---|---|---|
| Ограничение доступа к журналу | роль `multiagent_app` (только INSERT / SELECT), REVOKE UPDATE/DELETE/TRUNCATE | `\dp pipeline_audit` |
| Контроль целостности | SHA-256 `prompt_hash` / `completion_hash`; OpenTimestamps (опц.) | `ots verify <hash>.ots` |
| Учёт операций | append-only `pipeline_audit` (17 столбцов в 4 группах) | `SELECT COUNT(*) FROM pipeline_audit` |
| Изоляция контура | air-gapped-режим (Ollama + локальный PostgreSQL) | `docker network inspect` |
| Контроль человека | `ApprovalGate`; `ExplainerAgent`; поле `subtask_why` | UI-блок `TaskExplanation` |
