# Демо-режим

> Изображение экрана прилагается отдельным файлом. Ниже — текстовое описание и анализ.

## Что показано

**Демо-режим** (`Demo`) воспроизводит ранее записанный прогон без реальных
LLM-вызовов. На экране — выбор одного из **8 шаблонов задач** (по числу
поддерживаемых типов операций, например `add_entity`, `modify_entity`,
`add_service` и т. д.). После выбора шаблона интерфейс проигрывает записанную
последовательность этапов пайплайна так, как если бы выполнялся реальный прогон:
обновляются метрики, появляются diff и `TaskExplanation`.

## Анализ

Демо-режим решает две задачи. Во-первых, он позволяет демонстрировать
возможности системы без затрат на LLM-вызовы и без доступа к внешним моделям —
полезно для презентаций и обучения. Во-вторых, воспроизведение записанного
прогона детерминировано, что делает демо-режим стабильным и предсказуемым.
Восемь шаблонов покрывают весь набор поддерживаемых типов задач, давая зрителю
представление о диапазоне применимости системы. Важно: в режиме `Demo` реальная
кодогенерация не выполняется — это воспроизведение, а не новый прогон.
