**Назначение документа** — полная детализация 15 производственных задач плана улучшений системы **AI Junior Developer**, выполненных в марте — апреле 2026 г. Документ детализирует 15 производственных задач плана улучшений: приведены сводная таблица задач, журнал замечаний и полные карточки.

Арифметика раскрытия: 5 пилотных замечаний развернулись в 7 задач (замечание о KnowledgeBundle дало 3 связанные задачи — № 3, 9, 10), 3 аудитных — в 3 задачи, 5 проактивных — в 5 задач (итого 15).

|---|---|

### Задача 7 — Slim Coder + per-subtask RAG

|   |   |
|---|---|
| Источник | проактивная задача |
| Внутреннее обоснование | Промпт Coder содержит большую static-часть (системные правила, шаблоны EDIT-блоков, примеры — около 80 % промпта по объёму) и небольшую dynamic-часть (текущая подзадача, RAG-примеры — около 20 %). Без разделения и кэширования каждый LLM-вызов оплачивает все 100 % промпта; RAG-выборка примеров на уровне задачи (а не подзадачи) включает нерелевантные паттерны. |

### Задача 12 — Расширение Roslyn-графа до полного покрытия типов задач

|   |   |
|---|---|
| Источник | проактивная задача |

Число эндпоинтов 30 → около 50 (OpenAPI paths 30 → 48); покрытие типов задач 5 / 10 → 10 / 10. |

|---|---|

### Задача 15 — Compliance-аудит (append-only + replay)

|   |   |
|---|---|
| Источник | проактивная задача |
| Внутреннее обоснование | Развёртывание системы в регулируемом сегменте (банки, КИИ) требует полного журналирования LLM-вызовов с побайтовым воспроизведением — формальное требование Приказа ФСТЭК № 235 § II.2 и закрепляющее соответствие 152-ФЗ ст. 19 (учёт обращений к ПДн) + 187-ФЗ ст. 9 п. 3 (целостность регистрационных записей). До улучшения системы append-only-журнала и replay-функциональности не существовало. |
| Реализованный механизм | Append-only-журналирование на уровне СУБД через ограниченную Postgres-роль с явным `REVOKE UPDATE/DELETE/TRUNCATE`; SHA-256 хеши записей; опциональная нотаризация в Bitcoin через OpenTimestamps; CLI `multiagent_replay` для побайтового воспроизведения. |
