# Страница `/compliance` — аудит и воспроизведение

> Изображение экрана прилагается отдельным файлом. Ниже — текстовое описание и анализ.

## Что показано

Страница **`/compliance`** — панель аудита прогонов для compliance-офицера.
На экране представлены:

- **Журнал `pipeline_audit`** (append-only) с записями LLM-вызовов, отфильтрованными по `run_id`: видны `sequence`, `agent_name`, `model_name`, метка времени и длительность.
- Хеши **SHA-256** (`prompt_hash` / `completion_hash`) для каждой записи — доказательство неизменности.
- Панель **`ReplayPanel`** — побайтовое воспроизведение (byte-identical replay) LLM-вызовов прогона. На облачном провайдере (Anthropic) воспроизводимость составляет **2 из 4 (50 %)** из-за ограничений детерминизма на стороне провайдера; для локального режима (Ollama + фиксация `seed`) запланировано **100 %** (задача R10).
- Опциональный блок **OpenTimestamps** — криптографическая нотаризация хешей для дополнительной защиты от подмены постфактум.

## Анализ

Экран — центральный артефакт для проверки регулятором. Сочетание append-only-журнала,
SHA-256-хешей и воспроизведения даёт независимо проверяемый след: аудитор может
убедиться, что запись не менялась (хеш + OpenTimestamps) и при необходимости
воспроизвести вызов байт-в-байт. Показатель 50 % на Anthropic честно отражает
ограничение облачного провайдера и мотивирует локальный air-gapped-режим, где
фиксация `seed` обеспечивает полную воспроизводимость. Выгружаемый пакет
`audit-{run_id}.json` предъявляется регулятору без доступа к исходному коду агентов.
