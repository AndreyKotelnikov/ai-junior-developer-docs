# Принципы трассировки артефактов факторного эксперимента

Документ фиксирует принципы прозрачной трассируемости результатов факторного эксперимента: каждый числовой результат соотносится с конкретным шагом методики и конкретным артефактом (датасетом, конфигурацией, скриптом, отчётом). Связь «артефакт → шаг методики → результат» должна быть очевидна.

## Категории артефактов

Артефакты разделены на четыре категории:

- **Данные** — тестовый набор и сырые результаты прогонов;
- **Конфигурации** — описания развёртывания стендов;
- **Код** — скрипты прогона и расчёта;
- **Отчёты и методика** — агрегированные результаты и методические документы.

## Цепочка верификации любого результата эксперимента

Любой числовой результат факторного эксперимента верифицируется по единой цепочке:

1. **Идентификация результата** (например, «H4 паритет 84,85 %»).
2. **Поиск шага методики и артефакта** в trace-таблице артефактов.
3. **Проверка по `docs/technical/stats_report.md`** — агрегированный отчёт по 5 гипотезам с p-значениями и эффект-сайз.
4. **Контрольный пересчёт** через `docs/technical/compute_stats.py` на сырых данных `docs/experiment/test_results/stand-{A–I}-run-{1–5}.md`.

Каждое звено цепочки доступно для независимой проверки рецензентом.

## Принципы привязки артефактов

- **Один артефакт — одна категория**: каждый артефакт принадлежит только одной из четырёх категорий, без двойной классификации.
- **Один артефакт — один ключевой шаг методики**: для каждого артефакта в trace-таблице указан один шаг методики, к которому он относится (вспомогательные ссылки могут быть из других шагов).
- **Один артефакт — один измеримый результат**: для каждого шага методики указан измеримый результат — конкретное численное значение или конкретный набор записей.
- **Все числовые значения получены единым запуском**: значения в `stats_report.md` получены единым запуском `compute_stats.py` на 675 наблюдениях; значения в `stats_output_robustness.json` — единым запуском `compute_robustness_stats.py` (RNG seed = 42).

## Принцип воспроизводимости артефактов

Воспроизводимость обеспечена через:

- **Git-репозиторий с зафиксированным коммитом** — все скрипты и конфигурации в одной фиксированной версии;
- **Docker Compose-конфигурации 9 стендов** — явная фиксация значений факторов (Architecture × RAG × Graph × Model);
- **Зафиксированные параметры инференса** — `temperature = 0` на всех стендах;
- **RNG seed = 42** — для bootstrap-методов робастности.

Ограничение воспроизводимости: побайтовая идентичность отдельного прогона не гарантируется (Anthropic не гарантирует детерминизм при `temperature = 0,0`; параметр `seed` в Ollama в текущей реализации не фиксируется — задача R10 реестра).
