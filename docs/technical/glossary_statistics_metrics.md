# Глоссарий статистических и метрических терминов

Справочник 15 терминов, употребляемых в материалах факторного эксперимента. Каждое определение содержит формулу (где применимо), словесное определение и пример из факторного эксперимента (675 наблюдений, 9 стендов × 5 повторов × 15 задач).

## Авторские метрики

### Pass@1 (упрощённая, через успешную компиляцию)

Доля прогонов с успешным `dotnet build`. Полное определение и формула — `docs/technical/metrics_methodology.md`.

- Пример: стенд E — 90,67 % (68 успешных прогонов из 75).
- Отличие от классического Pass@1 (Chen et al., 2021) — не включает прохождение функциональных unit-тестов; функциональная корректность вынесена в самостоятельную метрику Completeness.

### Completeness

Полнота охвата изменений: доля файлов из F_required, фактически модифицированных системой.

- Формула: `mean(|F_actual ∩ F_required| / |F_required|)` по 75 прогонам стенда.
- Пример: стенд G — 62,44 %.

### TimeToSuccess

Медиана времени успешного прогона в секундах.

- Формула: `median(время прогонов с pass1 = 1)`.
- Пример: стенд G — 464,9 с.
- На стенде A (134 с) — «время до неудачи» single-agent (Pass@1 = 1,33 %), не показатель производительности.

### σ Pass@1

Выборочное стандартное отклонение Pass@1 по 5 повторам стенда (ddof = 1 — несмещённая оценка с делителем N − 1).

- Пример: стенд B — 5,58 %.
- Пороги: σ < 5 п.п. — стабильный стенд; 5 ≤ σ < 8 — умеренная вариативность; σ ≥ 8 — нестабильный.

### FSR (False Success Rate)

Доля прогонов с успешной сборкой при Completeness = 0 — критерий честности, фиксирует «формальный успех при полном промахе по адресации».

- Пример: стенд G — 0,00 % (без скрытых сбоев адресации); стенд B — 22,67 % (без графа адресация массово ошибочна).

### ARR (приведённый resolved rate)

Приведённая метрика для сравнения с внешними бенчмарками: `mean(b · c)`, где `b` — признак успешной сборки прогона (0/1), `c` — Completeness прогона.

- Аппроксимация через агрегаты: ARR ≈ Pass@1 × Completeness (даёт нижнюю консервативную оценку при положительной ковариации).
- Пример: стенд E — 58,18 % (формальное mean(b·c)); 54,12 % (агрегатная аппроксимация Pass@1 × Completeness).

## Статистические инструменты

### *p*-значение

Вероятность увидеть наблюдаемую или более экстремальную разность, если истинной разности нет (при условии нулевой гипотезы).

- Пример H1: *p* = 0,000061 — практически невозможно, что A vs C совпало случайно.

### *d* Коэна (*d_h*, *d_z*)

Размер эффекта в стандартных отклонениях.

- *d_h* — для разности пропорций, использует арксинус-преобразование для выравнивания шкалы: `d_h = 2·arcsin(√p₁) − 2·arcsin(√p₀)`.
- *d_z* — для парных разностей: `d_z = mean(diff) / std(diff, ddof=1)`.
- Шкала Cohen 1988 (Cohen, 1988): ≥ 0,2 — малый, ≥ 0,5 — средний, ≥ 0,8 — крупный.
- Примеры: H1 *d_h* = 1,98 (крупный); H3 *d_z* = 0,79 (крупный); H2b *d_z* = 0,26 (малый).

### Реализованная мощность (1 − β)

Вероятность обнаружить эффект данной величины при текущей выборке, если эффект действительно существует.

- Целевой уровень — 0,80 (Cohen, 1988).
- Примеры: H1 — 1,00 (поймает почти наверняка); H3 — 0,99; H4 — 0,05 (на грани шума).

### Парный критерий Уилкоксона (Wilcoxon signed-rank)

Непараметрический тест на сдвиг распределения парных разностей относительно нуля. Не требует нормальности; устойчив на малой выборке.

- В работе: применяется как primary тест для **H1, H2a, H2b, H4** на 15 задачах. **Для H3 primary — mixed-effects модель** (см. `docs/technical/compute_h3_mixed.py`) с учётом кластеризации 45 разностей по 15 задачам; one-sample Wilcoxon на 45 объединённых разностях — summary-индикатор консистентности, p = 5,8 × 10⁻⁵ (но завышен из-за допущения независимости 45 разностей, см. ICC = 0,74).
- Реализация: `scipy.stats.wilcoxon` с параметрами `zero_method='wilcox'`, `correction=False`, `alternative='two-sided'`, `method='auto'`.

### Поправка Холма (последовательная Бонферрони)

Контроль накопления ошибки I рода при семействе гипотез. При 5 гипотезах базовый порог Бонферрони: α_adj = 0,05 / 5 = 0,01. Холм мягче и даёт большую мощность через последовательную проверку с прогрессивно ослабляемым порогом (0,0100 → 0,0125 → 0,0167 → 0,0250 → 0,0500).

### TOST (Two One-Sided Tests)

Корректный тест для гипотезы эквивалентности: проверяет, лежит ли нижняя граница одностороннего ДИ выше порога эквивалентности. Schuirmann, 1987; Lakens, 2017.

- В работе: применяется к H4 (паритет on-prem vs cloud).
- Результат: нижняя граница 95 % ДИ на отношение Pass@1(H)/Pass@1(G) = 0,6575 > 0,65 (non-inferiority на уровне 0,65 формально установлена; на уровне 0,75 — нет: 0,6575 < 0,75).

### Bootstrap (бутстреп)

Метод ресэмплинга для оценки распределения статистики. В работе — 10 000 ресэмплов (RNG seed = 42); даёт эмпирическую вероятность направления эффекта без зависимости от дихотомии «p < α». Efron, 1979.

- Пример: P(паритет ≥ 0,75) = 0,806 для H4; P(эффект > 0) = 0,845 для H2b.

### 95 % ДИ Уилсона (Wilson score interval)

Устойчивый доверительный интервал для долей; учитывает асимметрию вблизи границ 0 и 1, в отличие от нормального приближения. Newcombe, 1998; z = 1,96 для 95 %.

- Пример: стенд E Pass@1 = 90,67 %, 95 % ДИ Уилсона = [81,97; 95,41] на n = 75 прогонах.

### FWER (Family-Wise Error Rate)

Вероятность хотя бы одной ошибки I рода в семействе гипотез. Поправки Бонферрони / Холма контролируют FWER ≤ 0,05.
