# Угрозы валидности результатов факторного эксперимента

Классификация угроз валидности по канонической схеме Cook & Campbell, 1979; адаптация для SE-экспериментов — Wohlin et al., 2012; современная актуализация — Ralph et al., 2021.

## Таблица — Угрозы валидности и применённые митигации

| Тип валидности | Угроза | Митигация в эксперименте | Остаточный риск |
|---|---|---|---|
| **Внутренняя** | Стохастичность LLM; конкуренция за VRAM | 5 повторов; `temperature = 0`; серийный планировщик Ollama | Anthropic не гарантирует полный детерминизм; параметр `seed` в Ollama в текущей реализации не фиксируется; σ Pass@1 = 5,6–8,2 п.п. |
| **Внешняя** | Один проект; 15 задач одного типа `modify_entity` (26 % MR-diff корпоративного GitLab); срез 26.03.2026 | Enterprise-масштаб (1 559 файлов); стратификация задач 5 + 5 + 5 по сложности | Перенос на другие ORM / языки и другие типы задач — задачи R3, R8 |
| **Конструктная** | Pass@1 ≠ функциональная корректность; Completeness ≠ Pass@1; эталон F_required формирует автор системы (потенциальный bias) | Раздельные метрики; FSR отдельно; F_required формируется из git-диффа ручной реализации (зафиксирован до прогонов) | Функциональная корректность — задача R3; двойная аннотация эталона на пилоте — задача R8 |
| **Выводов** | Множественное тестирование 5 гипотез; FWER при положительной корреляции тестов | Бонферрони (α_adj = 0,01); Холм; неперекрытие 95 % ДИ Уилсона | При положительной корреляции тестов FWER ≤ 0,05 — поправка Холма даёт верхнюю оценку (защита консервативна) |

## Контекст применения

- **Внутренняя валидность** подкреплена 5 повторами и `temperature = 0`. Параметр `seed` Ollama в текущей реализации не фиксируется — задача R10.
- **Внешняя валидность** ограничена `modify_entity` и одним проектом; перенос — задачи R3 (xUnit-сюита T01..T25) и R8 (пилот на стороннем .NET-проекте).
- **Конструктная валидность** обеспечена раздельными метриками Pass@1 / Completeness / FSR / ARR.
- **Валидность выводов** — поправка Холма + ДИ Уилсона + неперекрытие интервалов теста и baseline для формально подтверждаемых гипотез.

Для вторичных гипотез H2b и H4 дополнительно применена многометодная валидация (TOST, bootstrap, стратификация) — `docs/technical/robustness_methodology.md`.
