# Шлюз утверждения `ApprovalGate`

> Изображение экрана прилагается отдельным файлом. Ниже — текстовое описание и анализ.

## Что показано

Баннер **`ApprovalGate`** появляется на вкладке `План` перед началом
кодогенерации, когда включено требование утверждения плана человеком. Он
предъявляет пользователю план декомпозиции от `Orchestrator` — список подзадач,
каждая с полем `why` (обоснование «зачем»).

Пользователю доступны **три действия**:

1. **Одобрить** — план принимается, запускается кодогенерация.
2. **Запросить коррекции** — пользователь отправляет правки через plan-Editor; план пересобирается.
3. **Отклонить** — прогон останавливается.

Решение пользователя и обоснования подзадач фиксируются в поле **`subtask_why`**
журнала `pipeline_audit`.

## Анализ

`ApprovalGate` — узловая точка контроля человека над ИИ (Указ Президента РФ
№ 490). Принципиально, что кодогенерация **не стартует** без явного утверждения:
ни одна правка кода не выполняется «молча». Запись `subtask_why` в append-only-журнал
превращает акт человеческого контроля в проверяемый аудиторский артефакт — позднее
compliance-офицер может доказать, что изменение было санкционировано человеком, и
с каким обоснованием.
