# Паспорт корпуса для RAG

Документ описывает корпус исторических правок, используемый для
retrieval-augmented generation (RAG) — поиска проектного контекста,
передаваемого агенту-кодеру.

## 1. Состав и объём

| Атрибут | Значение |
|---|---|
| Содержание | пары MR-diff (merge request diff) — образцы исторических правок корпоративного кода |
| Объём | 1 154 пары MR-diff |
| Период | 2023–2025 гг. |
| Источник | внутренний корпоративный GitLab |
| Покрытие типов задач | 10 типов (add_entity, modify_entity, add_service, modify_service, bug_fix, filtering_sorting, import_export, refactoring, config_infra, ui_view) |
| Способ подготовки | векторизация и индексация в Qdrant; модификации содержимого правок не вносились |

Доля задач модификации существующих сущностей (`modify_entity`) — наибольшая среди
десяти типов: 26,0 % корпуса (300 из 1 154). На этом типе и проводится факторный
эксперимент.

## 2. Индексация и хранилище

Корпус индексируется в векторной базе данных Qdrant. Для каждой пары MR-diff
строится векторное представление (эмбеддинг) текста бизнес-требования и связанного
кода; параллельно поддерживается полнотекстовый индекс для лексического поиска.

| Компонент | Реализация |
|---|---|
| Модель эмбеддингов | `intfloat/multilingual-e5-base` (многоязычная, поддержка русского) |
| Векторное хранилище | Qdrant |
| Лексический поиск | BM25 |
| Объединение результатов | Reciprocal Rank Fusion (RRF) |
| Переранжирование | cross-encoder reranker |

## 3. Гибридный поиск

Поиск релевантных примеров — гибридный: семантический (по эмбеддингам E5) и
лексический (BM25) ранжированные списки объединяются методом RRF, после чего
кандидаты переранжируются cross-encoder-моделью. Итоговое качество ранжирования —
MRR (Mean Reciprocal Rank) = 0,765.

Из корпуса агенту передаётся фиксированное число ближайших примеров (0/3/5 в
зависимости от конфигурации эксперимента). Per-subtask-режим отбирает примеры под
конкретную атомарную подзадачу, отфильтровывая нерелевантные паттерны.

## 4. Роль в эксперименте

Корпус — один из трёх источников данных факторного эксперимента (наряду с целевым
приложением и тестовым набором задач). Он обеспечивает фактор RAG: число
передаваемых исторических примеров варьируется между стендами, что позволяет
изолировать вклад retrieval в качество кодогенерации.

Эмпирически установлено, что в изоляции от графовой адресации увеличение числа
примеров не повышает качество (контекстный шум). Поэтому исторический
классификатор-конвейер исключён из production, а корпус сохранён в Qdrant как
материал для семантического поиска в сочетании с co-change-графом.

## 5. Ограничения

Корпус сформирован из внутреннего корпоративного репозитория и непубличен. Перенос
RAG на сторонний проект требует пересборки корпуса и переиндексации под его
кодовую базу и конвенции.
