# Панель Settings (настройки запуска)

> Изображение экрана прилагается отдельным файлом. Ниже — текстовое описание и анализ.

## Что показано

Панель **Settings** управляет параметрами запуска пайплайна. Основные элементы:

- **Переключатель режима LLM**: облачный `API` (Anthropic Claude) ↔ локальный `Ollama` (air-gapped). Для значимых объектов КИИ рекомендуется Ollama-режим.
- **Тогглы 16 этапов пайплайна** — каждый из 16 этапов конвейера можно включить/выключить; среди них — переключатель «Утверждать план перед кодогенерацией» (показывает `ApprovalGate`).
- **Выбор числа RAG-примеров** (0 / 3 / 5) и режима запуска (`Real` / `Demo`).
- Параметры маршрутизации моделей: **`ModelRouter`** (per-agent routing — разные модели разным агентам) и **Serial-Ollama Scheduler** (последовательное планирование вызовов к локальному Ollama, чтобы не перегружать GPU).

## Анализ

Панель сводит в одном месте все рычаги управления стоимостью, детерминизмом и
режимом приватности прогона. Ключевой для регулируемого контура — переключатель
`API` ↔ `Ollama`: он определяет, покидают ли данные периметр организации.
`ModelRouter` позволяет комбинировать модели (например, более сильную модель для
планирования и более лёгкую для кодирования), а Serial-Ollama Scheduler
обеспечивает корректную работу локальных моделей на ограниченном GPU. Тогглы
этапов дают тонкий контроль над тем, какие стадии пайплайна выполняются.
